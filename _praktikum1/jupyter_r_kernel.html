Jupyteri R kerneli muutmine

Kui Jupyter Anacondaga koos paigaldada, siis tuleb Anacondas kaasa oma R (vaikimisi versioon 3.6). See aga thendab, et kik paketid on vaja uuesti paigaldada, sest Anaconda-R asub mujal kui sinu arvutisse varem paigaldatud R. Kui sul oma arvuti RStudios kik ttab aga Jupyteris ei tta, siis vta ette jrgmised sammud (kui ks variant ei tta, siis proovi jrgmist):

Samm nr 1

Esimese sammu tegemiseks on mitu varianti. Alusta esimesest ja kui see ei tta, siis proovi jrgmist. Kui saad vastuse stiilis

package IRkernel successfully unpacked and MD5 sums checked

Siis vid edasi minna sammu 2 juurde.

Variant 1

Kivita RStudios jrgmine ksk:

install.packages('IRkernel')

Variant 2

Kivita RStudios jrgmised ksud:

# Kui pakett 'devtools' on puudu
install.packages("devtools")

devtools::install_github("IRkernel/IRkernel")

Variant 3

See on ainlt Windowsi kasutajatele. Muude operatsioonissteemidega loodan kellegi abile.

  1. Lae alla ja paigalda RTools. Kui oled varem juba paigaldanud, siis jta vahele.
  2. Lae alla IRkernel zip-fail Githubist.
  3. RStudios kivita ksud (NB! Sea tkaustaks asukoht, kuhu laadisid .zip faili)
# Sea tkaustaks asukoht, kuhu laadisid .zip faili
setwd("C:/Users/<kasutaja>/Downloads")

install.packages("IRkernel-master.zip", repos = NULL)

Loodetavasti ks neist variantidest ttas.

Samm nr 2

Tee kindlaks kuhu kausta oled paigaldanud R-i. Niteks

Seejrel kivita Anaconda:

  1. Mine Anacondas Environments, kivita oma keskkond, kus jookseb ka R (Kodutde juhendis panime nimeks AndeteadusR).
  2. Vajuta rohelisele kolmnurgale ja vali Open Terminal.
  3. Trki sisse ksk cd ja oma Ri asukoht. Niteks cd "C:\Program Files\R\R-4.0.4\"
  4. Kivita ksk cd bin ja seejrel R.exe
  5. Selle peale peaks tulema tuttav R-i teavitustekst. Kivita ksk IRkernel::installspec(). Tulemuse peaks olema midagi sarnast:

Kui see ei nnestu (saad veateate stiilis failed to create process), siis kivita Anaconda administraatori igustega (Run as Administrator) ning proovi uuesti.

Kivita Jupyter (kui oli varem lahti, siis taaskivita). Ava mni Jupyter notebook vi tee lihtsalt thi notebook (R kerneliga) ja kivita ksk R.Version()$version.string. Kontrolli, kas see on seesama R, mis RStudios (Tools -> Global Options -> R version).

Vid veel testida, et proovi kivitada Jupyteris mni pakett mida sul kindlasti ei ole, nt. library(pack). Vastuseks peaksid saama Error in library(abd) : there is no package called pack ja see peabki nii olema. Siis mine RStudiosse ja jooksuta install.packages('pack'). Eduka paigalduse jrgselt proovi Jupyteris uuesti antud paketti sisse lugeda ja seekord peaks see nnestuma.
