Andmete automatiseeritud eraldamine veebist

Veebis on aina rohkem andmeid struktureerimata kujul. Niteks ilmaennustusi, korterite hindu, geograafilist infot, sotsiaalvrgustike andmeid, valimistulemusi, aktsiahindu vi ajalehe artikleid ei ole alati saadaval csv vi Exceli failina. Alati on vimalik andmeid manuaalselt kopeerida, aga see on tlikas, aeganudev ja vigaderohke. Veebilehtedel oleva info automatiseeritud eraldamiseks on suuresti kaks vimalust:

  1. Rakendusliidesed (web API) - struktureeritud http pringud tagastavad andmed (ldjuhul) JSON vi XML formaadis.

Paljud rakendusliidesed nuvad autentimist, mille kigus tuleb vastavale saidile kasutaja registreerida. Seejrel saab kasutaja omale unikaalse vtme, mis tuleb lisada tehtavale http pringule. ldiselt on populaarsemad rakendusliidesed hsti dokumenteeritud ning juhendit jrgides saab andmed ilusti ktte. Siiski leidub ka tielikult avatud rakendusliideseid, mis autentimist ei nua.

Erinevaid rakendusliideseid vib leida niteks siit vi siit.

NB! Mngureeglid:

  1. Vaata, kas andmed on struktureeritud kujul olemas vi on veebilehel olemas API (kui on, siis kasuta neid ja ra kraabi).
  2. Vaata ja austa veebilehe robots.txt faili (nt ut.ee/robots.txt). See mrab, milliseid kohti veebilehest lubatakse robotil protsessida.
  3. Kraabi ainult neid andmeid, mida vajad.
  4. Oota teatud aeg (niteks 1 sekund) prast iga uut pringut.

Veebilehtede lesehitusest

Et tegeleda andmete kraapimisega veebist, on vaja omada levaadet sntaksist, millega veebilehti koostatakse. Seega tutvume HTML-i phitdedega. Alustame nitest:

<!DOCTYPE html>
<html>
<head>
  <title>Page Title</title>
</head>

<body>
  <h1>This is a Heading</h1>
  <p>This is a paragraph.</p>
</body>

</html>

Et nha tulemust, vid levaloleva koodi salvestada .html failina ning avada brauseris. Alternatiivina kliki siia.

Lhidalt kokku vttes: HTML koosneb siltidest (tags), mis mrgitakse smbolite < > vahele ning mille abil kirjeldatakse veebilehe struktuuri.

Rohkem infot HTML siltide kohta leiad siit.

HTML mrab veebilehe ldise struktuuri. Et muuta teksti vrvi ja muud stiili, selleks kasutatakse CSS-i.

Vaata niteid:

  • Minimaalne nide
  • Kuidas kasutada CSS id-sid
  • Kuidas kasutada CSS klasse

Andmete kttesaamise idee seisnebki selles, et veebilehe lhtekoodist leiame lesse vajaliku sildi ning eraldame vastava vrtuse. CSS-i mratud klassid ja id-d, aitavad meil tihti hlpsamini vajaliku info asukohta mrata.

Paketi rvest minimaalne nide

rvest on Hadley tehtud pakett veebilehtedelt andmete kraapimiseks.

HTML

Minimaalne nide, kuidas eelneval pildil nidatud HTML lhtekoodist eraldada pealkirja ja liku.

library(rvest)

html_source = '
<!DOCTYPE html>
<html>
<head>
  <title>Page Title</title>
</head>

<body>
  <h1>This is a Heading</h1>
  <p>This is a paragraph.</p>
</body>

</html>
'

page = read_html(html_source)

# pealkirja eraldamine
page %>% 
  html_node("h1") %>%
  html_text()
## [1] "This is a Heading"
# ligu eraldamine
page %>% 
  html_node("p") %>%
  html_text()
## [1] "This is a paragraph."

Funktsioonile read_html vib ette anda kas veebilehe urli, faili asukoha vi sne.

XML

XML on mrgistuskeel, mille eesmrgiks on struktureeritud info jagamine. XML koosneb kasutaja defineeritud siltidest (tags), mis mrgitakse smbolite < > vahele ja mille abil kirjeldatakse andmete struktuuri. Jrgnevalt on toodud minimalistlik nide, kuidas XML failist eraldada infot.

library(rvest)

xml_source = '
<?xml version="1.0" encoding="UTF-8"?>
<inimesed>
  <inimene>
    <nimi>Mati</nimi>
    <vanus>25</vanus>
  </inimene>
  <inimene>
    <nimi>Kati</nimi>
    <vanus>40</vanus>
  </inimene>
</inimesed>
'

page = read_html(xml_source)

# vanuste eraldamine
page %>% 
  html_nodes("inimene") %>%
  html_node("vanus") %>%
  html_text()
## [1] "25" "40"

Funktsiooni read_html asemel vib kasutada funktsiooni read_xml vi html_node asemel xml_node, kuid vastavad XML-ekvivalendid kutsuvad vlja vastavad html funktsioonid, seega ei ole tegelikult vahet, kumba varianti kasutada.
